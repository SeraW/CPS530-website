Install Materalize

Step 1: Getting Started

You can install Materialize by going directly to the website, or by using NPM. Simply run npm install materialize-css@next if you're on Linux.

Install From Here
  • Materialize

Step 2: Directory Setup

Your initial directory will look something like this. materialize.css and materialize.jss both come with prepackaged assets that are ready to use.

Step 3: HTML Setup 

             
<!DOCTYPE html>
<html>
    <head>
    <!--Import Google Icon Font-->
    <link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">
    <!--Import materialize.css-->
    <link type="text/css" rel="stylesheet" href="{{ url_for('static', filename='css/materialize.min.css') }}"  media="screen,projection"/>
    <link type="text/css" rel="stylesheet" href="{{ url_for('static', filename='css/page1.css') }}">

    <!--Let browser know website is optimized for mobile-->
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    </head>

Install Heroku

Step 1: Download and make an account

Heroku is a PaaS that allows users to develop their own apps on its cloud platform. We will be hosting our web app on Heroku, at the low, low cost of $0! You can directly link your Git repo with Heroku.

Sign up here Download here
Step 2: Directory setup pt2

Flask and Heroku will recognize specific directory setups. The Procfile needs to be in your app's root directory, as it will not work anywhere else. Both requirements.txt and wsgi.py are also usually located at the root. The static folder houses css, img, and js material, and Flask is able to access them using functions like url_For. 


heroku login
git add --all
git commit -m "first commit"
git push
Step 3: Push to Git

Push your files to Git. If you have the option enabled, Heroku should automatically sync. Your app is now deployed!
